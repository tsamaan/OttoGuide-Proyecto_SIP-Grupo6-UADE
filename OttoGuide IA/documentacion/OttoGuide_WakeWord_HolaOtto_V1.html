Concepto

¿Qué es un wake word y cómo funciona?

Antes de entender el problema, entender el concepto es clave para tomar decisiones correctas.

🎤 Wake word dedicado
Un modelo de Machine Learning ultraliviano que escucha el micrófono 24/7 y detecta una frase específica. Consume muy pocos recursos (~50MB RAM) porque solo hace una cosa: detectar esa frase. Ejemplos: "Alexa", "Hey Siri", "OK Google".
🔁 El flujo correcto
El wake word activa el pipeline pesado. Mientras no se detecta la frase, Whisper (STT), el LLM y el TTS están apagados. Esto ahorra recursos críticos del Jetson y alarga la vida útil de la batería del robot durante el tour.
🧠 Cómo se entrena
Un modelo de wake word se entrena con muestras positivas (la frase objetivo) y muestras negativas (ruido de fondo y palabras similares). Cuantas más muestras variadas, más robusto es el modelo.
📦 OpenWakeWord
La librería elegida para OttoGuide. Corre en Python, es open source, compatible con ARM (Jetson), y permite crear modelos personalizados. El problema es que el proceso de entrenamiento requiere datos sintéticos masivos.

Solución actual · MVP

Solución para el MVP — detección via Whisper

Para la demo del miércoles se implementa una solución basada en Whisper que ya está funcionando. Detecta "Hola Otto" sin entrenamiento previo.

🎯 Pipeline MVP — Whisper como detector de wake word
En lugar de un modelo dedicado, el orquestador graba fragmentos cortos de audio cada 3 segundos y los manda a Whisper. Si la transcripción contiene "otto", activa el pipeline completo. Es menos eficiente en recursos pero funciona perfectamente para la demo y ya está probado.
pythonservices/core/main.py — lógica de wake word via Whisper
# Loop principal de detección import subprocess, requests, time def grabar_fragmento(duracion=3): subprocess.run([ "arecord", "-d", str(duracion), "-D", "plughw:1,0", "-f", "S16_LE", "-c", "2", "-r", "16000", "/tmp/fragmento.wav" ]) def transcribir(): r = requests.post("http://stt:9000/asr", files={"audio_file": open("/tmp/fragmento.wav", "rb")}, data={"language": "es"}) return r.text.lower() while True: grabar_fragmento() texto = transcribir() if "otto" in texto: print("Wake word detectado! Activando pipeline...") activar_pipeline() # llama a STT completo → LLM → TTS time.sleep(0.1)
⚠ Limitación del MVP: Graba en ciclos de 3 segundos — puede tardar hasta 3 segundos en detectar el wake word. Consume Whisper de forma continua en lugar de solo cuando es necesario. Aceptable para demo, no para producción.
Plan definitivo

Cómo crear "Hola Otto" correctamente

El proceso completo para entrenar un modelo de wake word personalizado con OpenWakeWord. Requiere más tiempo pero es la solución correcta para producción.

ℹ OpenWakeWord necesita que el modelo base sea entrenado por su pipeline oficial de generación de datos sintéticos. Las 14 muestras grabadas se usan como referencia para el verificador final, no como dataset de entrenamiento principal.
Paso 1 — Clonar el repositorio de entrenamiento
Obtener las herramientas de entrenamiento de OpenWakeWord
El entrenamiento completo no está en el paquete pip — está en el repositorio de GitHub.
bash
git clone https://github.com/dscripka/openWakeWord.git cd openWakeWord pip install -r requirements.txt --break-system-packages
Paso 2 — Instalar dependencias de generación de datos
Text-to-Speech para generar muestras sintéticas
El entrenamiento genera miles de muestras sintéticas de "Hola Otto" con diferentes voces, acentos y condiciones de ruido usando TTS. Requiere estas dependencias adicionales.
bash
pip install TTS piper-tts --break-system-packages # También necesita datasets de ruido de fondo # El script los descarga automáticamente (~2GB)
Paso 3 — Generar datos sintéticos de "Hola Otto"
Crear el dataset de entrenamiento
El script genera miles de variaciones de la frase con diferentes voces TTS en español. Esto puede tardar 1-2 horas dependiendo del hardware.
bashdesde el directorio openWakeWord/
python training/generate_training_data.py \ --target_phrase "Hola Otto" \ --output_dir data/hola_otto/ \ --n_samples 5000 \ --language es
Paso 4 — Entrenar el modelo base
Entrenamiento principal del wake word
Usa los datos sintéticos generados para entrenar el modelo neuronal. Tarda entre 30 minutos y 2 horas en CPU.
bash
python training/train.py \ --training_data data/hola_otto/ \ --model_name hola_otto \ --output_dir models/ \ --epochs 50
Paso 5 — Entrenar el verificador con las muestras reales
Afinar el modelo con voces reales grabadas
Acá es donde entran las 14 muestras que ya grabamos. El verificador aprende a reconocer "Hola Otto" con voces reales de personas reales.
pythontrain_verifier.py
from openwakeword import train_custom_verifier train_custom_verifier( positive_reference_clips="services/wake-word/samples/", negative_reference_clips="data/negative_samples/", output_path="services/wake-word/model/hola_otto_verifier.joblib", model_name="models/hola_otto.onnx" )
Paso 6 — Integrar al contenedor wake-word
Copiar los modelos al servicio Docker
Una vez entrenados, los archivos del modelo se copian a services/wake-word/model/ y se actualizan en el Dockerfile y el script listen.py.
bash
cp models/hola_otto.onnx services/wake-word/model/ cp models/hola_otto_verifier.joblib services/wake-word/model/ # Rebuild del contenedor docker compose up -d --build wake-word

Dato importante

Muestras negativas — qué son y cómo grabarlas

Para que el modelo sea preciso y no se active por error, necesita aprender también qué NO es "Hola Otto".

¿Qué son las muestras negativas?
Son grabaciones de audio que se parecen fonéticamente a "Hola Otto" pero no lo son. El modelo aprende a distinguir la frase exacta de palabras similares. Sin esto, el sistema se activa con cualquier palabra parecida.
